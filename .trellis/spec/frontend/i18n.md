# i18n (Paraglide)

> Executable contract for internationalization: Paraglide message catalog, locale fallback, menu-title dynamic resolution, BA error translation.

---

## 1. Scope / Trigger

Triggers when work touches: any user-facing `.tsx` text (JSX literals, alt, aria-label, placeholder, toast, form label) · `messages/{zh,en}.json` · `project.inlang/settings.json` · `src/components/layout/AppSidebar.tsx` menu label resolution · `src/lib/auth-errors.ts` · `src/emails/*.tsx` · adding/deprecating a locale.

Generated `src/paraglide/*` are **read-only** — regenerate via Vite plugin, never hand-edit.

---

## 2. Signatures

### Paraglide message exports (`#/paraglide/messages`)

```ts
import * as m from "#/paraglide/messages";

// Static access — TS verifies the key at compile time.
m.common_loading();                     // "加载中..."
m.email_subject_verify();               // "请验证你的邮箱"

// With params
m.email_invite_preview({ inviterName, organizationName });
```

Keys with dots (e.g. `menu.dashboard`) compile to string-named exports on the `m` namespace — **not** nested objects.

### Dynamic menu label resolver (`src/components/layout/AppSidebar.tsx`)

```ts
function resolveMenuLabel(title: string | undefined): string | undefined {
  if (!title) return undefined;
  if (title.startsWith("menu.") && title in m) {
    const fn = (m as unknown as Record<string, () => string>)[title];
    if (typeof fn === "function") return fn();
  }
  return title;   // fallback: operator-created menus keep literal titles
}
```

Only the `menu.` prefix triggers resolution. Titles without the prefix render verbatim — operators add menus via the admin UI without forcing them into the catalog.

### BA error translator (`src/lib/auth-errors.ts`)

```ts
export interface AuthErrorLike {
  code?: string | null;
  message?: string | null;
}

export function translateAuthError(error: AuthErrorLike | null | undefined): string;

// Usage:
const { error } = await authClient.signIn.email({ email, password });
if (error) toast.error(translateAuthError(error));
```

- Looks up `error.code` (uppercased) in static `CODE_TO_MESSAGE` map → Paraglide fns.
- Falls back to `error.message` (raw BA English) if code absent/unknown.
- Final fallback: `m.common_unknown_error()` — never returns empty/undefined.

## 3. Contracts

### Project config (`project.inlang/settings.json`)

```jsonc
{
  "baseLocale": "zh",
  "locales": ["zh", "en"],
  "plugin.inlang.messageFormat": {
    "pathPattern": "./messages/{locale}.json"
  }
}
```

- `baseLocale: "zh"` — all lookups fall back to `zh` when a locale is missing a key. `en.json` intentionally holds empty-value placeholders; the Chinese string renders until a translator fills in.
- `locales` **must** list every `messages/*.json`. Stray files (e.g. `de.json` from scaffold) must be deleted — Paraglide warns on compile otherwise.

### Rules for UI text

1. **No hard-coded CJK/EN in JSX/alt/aria/title/toast.** Every literal must be `m.xxx()`.
2. Add new keys to `messages/zh.json` **first**; Paraglide's type generation flags missing keys at call sites.
3. Parameterized copy uses `{name}` placeholders (`@inlang/plugin-message-format@4`):
   ```json
   "email_invite_preview": "{inviterName} 邀请你加入 {organizationName}"
   ```
   Generated TS fn requires `m.email_invite_preview({ inviterName, organizationName })`.
4. `messages/en.json` mirrors the zh key set (empty strings or real translations). Never delete a key from one locale without deleting from all.

### Menu `meta.title` convention

- Seeded menus store **i18n keys** starting with `menu.` (e.g. `menu.dashboard`, `menu.settings_organization`).
- Operator-created menus store plain strings. `resolveMenuLabel` is a no-op for those.
- Both coexist in one `Menu.meta.title` column — no schema split.

### Email templates

All copy in `src/emails/*.tsx` (subject, body, CTA, ignore-hint) goes through `m.email_*()`. Templates render during `sendEmail` server-side — Paraglide's static exports work identically on server and client.

## 4. Validation Matrix

| Condition | Expected |
|---|---|
| Key in `zh.json`, missing in `en.json` | Fallback to zh (per `baseLocale`); no TS error |
| Key missing in `zh.json` | TS error at `m.<key>()` call site (Paraglide fails compile) |
| Menu `meta.title` not starting with `menu.` | `resolveMenuLabel` returns literal → ok for operator menus |
| Menu `meta.title` starts with `menu.` but key missing | `in m` fails → renders raw key (e.g. `menu.ghost`) — visible bug, add key |
| BA error `{ code: "INVALID_EMAIL_OR_PASSWORD" }` | Returns `m.auth_error_invalid_email_or_password()` |
| BA error with unknown `code` + non-empty `message` | Returns raw BA English message |
| BA error without `code` + without `message` | Returns `m.common_unknown_error()` |
| Stray `messages/de.json` | Paraglide compile warning; build still produces `src/paraglide/messages.js` |
| Manual edit of `src/paraglide/*` | Lost on next `pnpm dev` / `pnpm build` (regenerated) |

## 5. Good / Bad Cases

### Good

```tsx
<Button>{m.common_save()}</Button>
toast.error(translateAuthError(error));
meta: { title: "menu.settings_organization" }
<Text>{m.email_verify_body()}</Text>
```

### Acceptable (operator-created menu)

```tsx
meta: { title: "业务日报" }  // resolveMenuLabel returns as-is; intentional
```

Non-technical operators add menus at runtime; forcing them into the catalog would defeat the feature.

### Bad

```tsx
<Button>保存</Button>                   // ❌ no m.xxx()
toast.error("邮箱或密码错误");          // ❌ use translateAuthError
<img alt="头像" />                       // ❌ aria/alt needs m.xxx()

// src/paraglide/messages.js
export const common_save = () => "saved";  // ❌ hand-edit — lost on next build
```

## 7. Wrong vs Correct

```tsx
// ❌ Hard-coded Chinese in JSX
<AlertDialog>
  <AlertDialogTitle>确认删除</AlertDialogTitle>
  <AlertDialogDescription>此操作不可撤销。</AlertDialogDescription>
  <AlertDialogCancel>取消</AlertDialogCancel>
  <AlertDialogAction onClick={onConfirm}>确认删除</AlertDialogAction>
</AlertDialog>
// Translator can't find strings; future en-US rollout needs a full JSX sweep;
// copy changes require code edits + redeploy.

// ✅ Paraglide messages
import * as m from "#/paraglide/messages";

<AlertDialog>
  <AlertDialogTitle>{m.confirm_dialog_default_title()}</AlertDialogTitle>
  <AlertDialogDescription>{m.confirm_dialog_irreversible_hint()}</AlertDialogDescription>
  <AlertDialogCancel>{m.common_cancel()}</AlertDialogCancel>
  <AlertDialogAction onClick={onConfirm}>
    {m.confirm_dialog_default_confirm()}
  </AlertDialogAction>
</AlertDialog>
```

Rationale: one catalog, one compile-checked call site, one-file diff to add `en`.
