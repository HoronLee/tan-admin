# Quality Guidelines

> Frontend quality gates and non-negotiable rules.

---

## Package Manager and Scripts

Use `pnpm` only. Do not use `npm` or `yarn` in this repo.

```json
// package.json
"scripts": {
  "dev":         "dotenv -e .env.local -- ...",
  "build":       "vite build && cp instrument.server.mjs .output/server",
  "test":        "vitest run",
  "check":       "biome check",
  "db:generate": "dotenv -e .env.local -- zen generate"
},
"pnpm": {
  "onlyBuiltDependencies": ["@sentry/cli", "esbuild", "lightningcss"]
}
```

## Lint and Format Baseline

Before commit, run `pnpm check`. Formatting and linting are unified in Biome:

```json
// package.json
"format": "biome format",
"lint":   "biome lint",
"check":  "biome check"
```

```json
// biome.json

"assist":     { "actions": { "source": { "organizeImports": "on" } } },

```

## Testing Requirements

Vitest for unit tests, React Testing Library for UI tests. New tests colocated as `*.test.ts(x)` or in `__tests__` layout.

```json

"vitest":                 "^3.0.5"
```

## Logging Rules in Frontend Surface

- `console.error` is acceptable for user-visible failure paths.
- Avoid `console.log` in reusable components and shared libraries — use the module logger or `reportError` instead.
- Never log BA session tokens, auth cookies, or password-like payloads.

```ts
// ✅ catch block in route components
try {
  await orpc.todos.create.call(input);
} catch (error) {
  reportError(error);          // frontend unified path (see backend/error-handling.md "Client-Side Consumption")
}

// ❌ demo-only console.log should not leak into shared components
onSubmit: ({ value }) => { console.log(value) }
```

## Environment and Secrets

- Development scripts require `.env.local` via `dotenv -e .env.local`.
- Keep secrets out of source; values exposed to client must use `VITE_` prefix.
- Server/client switch pair (`PRODUCT_MODE` / `VITE_PRODUCT_MODE`) **must** stay in sync — see `backend/product-modes.md`. Team 能力门控已改读 `organization.plan`（见 `backend/plan-gating.md`），不再用 env flag。

```json
// package.json
"dev":        "dotenv -e .env.local -- ...",
"db:push":    "dotenv -e .env.local -- zen db push",
"db:migrate": "dotenv -e .env.local -- zen migrate"
```

```ts
// src/env.ts
clientPrefix: 'VITE_',
client: {
  VITE_APP_TITLE:    z.string().min(1).optional(),
  VITE_PRODUCT_MODE: z.enum(["private", "saas"]).default("private"),
},
```

## Sentry Wiring Must Stay Enabled

Do not remove Sentry preload wiring from server startup commands:

```json
"dev":   "... NODE_OPTIONS='--import ./instrument.server.mjs' vite dev ...",
"start": "node --import ./.output/server/instrument.server.mjs .output/server/index.mjs"
```

```js
// instrument.server.mjs
Sentry.init({
  dsn: sentryDsn,
  sendDefaultPii: true,
  tracesSampleRate: 1.0,
  replaysSessionSampleRate: 1.0,
  replaysOnErrorSampleRate: 1.0,
})
```

## Accessibility Baseline

Any icon-only or context-poor interactive control must include assistive labels.

```tsx
// src/components/Header.tsx
<span className="sr-only">Follow TanStack on X</span>
<span className="sr-only">Go to TanStack GitHub</span>

// src/components/ThemeToggle.tsx / LocaleSwitcher.tsx
aria-label={label}
aria-pressed={locale === currentLocale}
```

**i18n gotcha**: all user-facing copy (including `sr-only`, `aria-label`, `alt`, placeholders, toast text) must route through Paraglide `m.xxx()`. See `frontend/i18n.md`.

## Hard NO Anti-Patterns

- Running `npm install` / `yarn add` for project dependencies.
- Cross-level relative imports when `#/*` alias is available.
- Editing generated files (`src/routeTree.gen.ts`, `zenstack/{schema,models,input}.ts`, `src/paraglide/**`).
- Hardcoding secrets in route/component code.
- Importing shadcn primitives from package names instead of local `src/components/ui/*`.
- Directly mutating store state objects — use immutable update patterns.
- Hard-coded Chinese/English in JSX — must go through `m.xxx()` (`frontend/i18n.md`).
- Using `reportError` for Better Auth client errors — use `translateAuthError` (`frontend/i18n.md` + `backend/error-handling.md`).

```ts
// ✅ shadcn primitives via local re-export
import { Button } from '#/components/ui/button'
import { Input }  from '#/components/ui/input'

// ✅ immutable store update
store.setState((state) => ({ ...state, firstName: e.target.value }))
```
