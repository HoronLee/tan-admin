# Quality Guidelines

> Frontend quality gates and non-negotiable rules.

---

## Package Manager and Scripts

Use `pnpm` only. Do not use `npm` or `yarn` in this repository workflow.

### Evidence

Source: script contract in `package.json:8-23`.

```json
"scripts": {
  "dev": "dotenv -e .env.local -- ...",
  "build": "vite build && cp instrument.server.mjs .output/server",
  "test": "vitest run",
  "check": "biome check",
  "db:generate": "dotenv -e .env.local -- zen generate"
}
```

Source: pnpm section in `package.json:85-93`.

```json
"pnpm": {
  "onlyBuiltDependencies": ["@sentry/cli", "esbuild", "lightningcss"]
}
```

## Testing Requirements

Use Vitest for unit tests and React Testing Library for UI tests. New tests should be added under `src/**` as colocated `*.test.ts(x)` or `__tests__` layout.

### Evidence

```json
"test": "vitest run"
```

Source: test dependencies in `package.json:71-72,79,83`.


## Logging Rules in Frontend Surface

- `console.error` is acceptable for user-visible failure paths.
- Avoid leaving `console.log` in reusable components and shared libraries.

### Evidence

Source: user-path error logging in `src/routes/demo/todos.tsx:43-45`.

```ts
} catch (error) {
  console.error('Failed to create todo:', error)
}
```

Source: demo-only `console.log` in route pages (`src/routes/demo/form.simple.tsx:24-27`, `src/routes/demo/form.address.tsx:36-40`).

```ts
onSubmit: ({ value }) => {
  console.log(value)
  alert('Form submitted successfully!')
}
```

## Environment and Secrets

- Development scripts require `.env.local` via `dotenv -e .env.local`.
- Keep secrets out of source; values exposed to client must use `VITE_` prefix.

### Evidence

Source: `.env.local` injection in `package.json:9,18-22`.

```json
"dev": "dotenv -e .env.local -- ...",
"db:push": "dotenv -e .env.local -- zen db push",
"db:migrate": "dotenv -e .env.local -- zen migrate"
```

Source: env typing in `src/env.ts:13,15-17`.

```ts
clientPrefix: 'VITE_',
client: {
  VITE_APP_TITLE: z.string().min(1).optional(),
},
```

## Accessibility Baseline

Any icon-only or context-poor interactive control must include assistive labels.

### Evidence

Source: `src/components/Header.tsx:27,41`.

```tsx
<span className="sr-only">Follow TanStack on X</span>
<span className="sr-only">Go to TanStack GitHub</span>
```

Source: `src/components/ThemeToggle.tsx:74`, `src/components/LocaleSwitcher.tsx:28`.

```tsx
aria-label={label}
aria-pressed={locale === currentLocale}
```

## Hard NO Anti-Patterns

- Running `npm install` / `yarn add` for project dependencies.
- Cross-level relative imports when `#/*` alias is available.
- Editing generated files (`src/routeTree.gen.ts`, `zenstack/*.ts`, `src/paraglide/*`).
- Hardcoding secrets in route/components code.
- Importing shadcn primitives from package names instead of local `src/components/ui/*`.
- Directly mutating store state objects.

### Evidence

Source: local shadcn import style `src/components/demo.FormComponents.tsx:5-12`.

```ts
import { Button } from '#/components/ui/button'
import { Input } from '#/components/ui/input'
```

Source: immutable update style `src/routes/demo/store.tsx:16-18`.

```ts
store.setState((state) => ({ ...state, firstName: e.target.value }))
```
